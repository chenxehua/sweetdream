## 1. 项目背景与目标

### 1.1 项目背景

#### 市场需求
- 中国0-14岁儿童约2.23亿人，65%的3-12岁儿童存在不同程度睡眠问题
- 儿童失眠发生率20%-30%，行为性失眠可持续至青少年期（54.5%）和成年（43.3%）
- 全球睡眠App市场11.24亿美元，CAGR 16%；中国儿童睡眠产品市场450-500亿元

#### 核心痛点
- **儿童侧**：入睡困难、夜醒频繁、焦虑恐惧导致睡眠问题
- **家长侧**：哄睡耗时长（平均1-2小时）、方法缺乏系统性、育儿焦虑

#### 现有方案缺陷
- 90%产品以白噪音+睡眠监测为主，功能同质化严重
- 未解决"焦虑-入睡困难"的根本心理链条
- 家长端支持不足，只关注孩子忽略家长焦虑
- App稳定性差、收费不透明

### 1.2 项目目标

#### 总体目标
针对3-12岁儿童，打造基于儿童心理学的一体化睡眠健康解决方案，帮助家庭建立科学睡前仪式、改善儿童睡眠质量、缓解家长育儿焦虑。

#### 核心价值主张
从"工具型白噪音"升级为"心理睡眠健康专家"，以CBT-I认知行为疗法为核心，结合睡前仪式引导和家长育儿指导，提供有临床依据的系统性干预方案。

#### 差异化定位
| 维度 | 传统方案 | 本项目 |
|------|---------|--------|
| 核心能力 | 白噪音播放 | 心理学干预+睡前仪式 |
| 关注对象 | 儿童端 | 儿童+家长双端 |
| 方法论 | 被动助眠 | 主动行为干预（CBT-I） |
| 内容深度 | 通用内容 | 分龄定制+情绪疏导 |

## 2. 建设范围与边界

### 2.1 核心建设范围

#### 用户群体
- **主用户**：3-12岁儿童家长
- **直接用户**：3-12岁儿童（在家长陪同下使用）

#### 功能范围
| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| 睡前仪式 | 睡前流程引导、仪式模板定制、进度追踪 | P0 |
| 音频内容 | 助眠故事、白噪音、冥想音乐、情绪疏导内容 | P0 |
| 家长端 | 孩子档案、设置管理、睡眠报告查看 | P0 |
| 睡眠追踪 | 入睡时间记录、睡眠质量评估、数据报告 | P1 |
| 家长指导 | 育儿知识、睡眠科普、专家课程 | P1 |
| CBT-I方案 | 行为疗法指导、认知重建练习 | P2 |
| 高级服务 | 专家咨询、社群互动 | P2 |

### 2.2 明确边界（Out of Scope）

- ❌ 睡眠硬件设备开发与销售
- ❌ 线下心理咨询/诊疗服务
- ❌ 儿童社交功能（保护隐私安全）
- ❌ 成人睡眠管理功能
- ❌ 医疗诊断与处方服务
- ❌ 跨境数据处理

### 2.3 技术边界

- **平台**：微信小程序（iOS/Android）
- **用户端**：儿童端（简化操作）、家长端（功能完整）
- **音频格式**：MP3、AAC
- **离线支持**：核心睡前仪式流程支持离线使用

### 4.2 前端技术选型

#### 微信小程序原生开发
| 项目 | 选型 | 理由 |
|------|------|------|
| 开发框架 | 微信小程序原生 | 稳定性最优、审核通过率高、API支持完善 |
| UI框架 | Vant Weapp / ColorUI | 组件丰富、风格统一、开发效率高 |
| 状态管理 | MobX-mini / Redux | 满足复杂页面状态管理需求 |
| 音频播放 | wx.createInnerAudioContext | 原生API，稳定性好 |
| 后台播放 | BackgroundAudioManager | 支持后台播放、定时关闭 |
| 动画 | CSS动画 / Lottie | 满足儿童端趣味动画需求 |

#### 备选方案：Taro/Uni-app
- **优势**：代码复用，可扩展至抖音/支付宝小程序
- **劣势**：审核风险增加，首屏加载略慢
- **建议**：MVP阶段使用原生，V1.0后评估是否迁移

### 4.3 后端技术选型

#### 微信云开发（TCB）
| 服务 | 说明 | 适用场景 |
|------|------|----------|
| 云函数 | Node.js运行环境 | 业务逻辑、API封装 |
| 云数据库 | MongoDB兼容JSON文档数据库 | 用户数据、内容数据存储 |
| 云存储 | 文件存储服务 | 音频文件、用户上传文件 |
| 静态网站托管 | 静态资源托管 | H5页面、活动页 |

#### 为什么选择微信云开发？
1. **低成本**：个人开发者免费额度充足（日均10000次云函数调用、5GB存储）
2. **免运维**：无需管理服务器，降低技术门槛
3. **高集成**：与微信登录、支付、订阅消息无缝对接
4. **合规性**：数据存储在国内，符合隐私法规要求

#### 备选：独立云服务器
- **适用场景**：用户量超过10万、需复杂业务逻辑
- **推荐配置**：2核4G云服务器（腾讯云/阿里云，年费约1000-2000元）
- **技术栈**：Node.js + Express / Python + Flask

### 4.4 音频存储方案

#### 方案一：微信云存储（推荐MVP）
- **优点**：免费额度充足（10GB）、CDN加速、与小程序无缝集成
- **缺点**：大文件存储成本随用户增长
- **适用**：音频文件总大小 < 50GB

#### 方案二：阿里云OSS / 腾讯云COS
| 方案 | 存储费用 | 流量费用 | 适用规模 |
|------|---------|---------|----------|
| 微信云存储 | 免费10GB，超出0.1元/GB/月 | 内网免费 | MVP初期 |
| 阿里云OSS | 0.12元/GB/月 | 0.5元/GB | 50GB+ |
| 腾讯云COS | 0.1元/GB/月 | 按量计费 | 50GB+ |

#### 音频文件规格
- **格式**：MP3（主流兼容）
- **码率**：64kbps（白噪音）/ 128kbps（故事）
- **单文件大小**：白噪音3-5MB，故事10-30MB
- **总内容库**：初期50GB，逐步扩展至200GB

### 6.2 重点风险应对

#### 风险1：合规风险（儿童数据隐私）

**风险描述**
- 涉及儿童用户，需遵守《儿童个人信息网络保护规定》
- 需建立完善的家长监护机制
- 内容需符合儿童心理发展特点

**应对措施**
1. **隐私设计**：默认不收集精确位置、生物特征等敏感数据
2. **家长授权**：14岁以下儿童需家长账号授权登录
3. **数据最小化**：仅收集功能必需数据，明确告知用途
4. **内容审核**：所有内容经专家审核，确保适合儿童
5. **法律合规**：聘请法律顾问审核隐私政策

#### 风险2：留存率风险

**风险描述**
- 睡眠类产品用户粘性挑战大
- 睡前仪式可能难以坚持

**应对措施**
1. **仪式养成设计**：游戏化元素、成就系统
2. **个性化内容**：基于孩子年龄、偏好推荐
3. **家长提醒**：科学的睡前提醒功能
4. **数据反馈**：睡眠改善可见，增强信心
5. **渐进引导**：CBT-I原理，循序渐进建立习惯

#### 风险3：内容专业性风险

**风险描述**
- 心理学内容需专业背书
- 用户可能质疑专业性

**应对措施**
1. **专家顾问**：签约1-2名儿童睡眠/心理学专家
2. **循证依据**：引用临床研究和权威指南
3. **内容标注**：明确标注内容来源和适用年龄
4. **用户教育**：帮助家长理解科学原理
5. **持续迭代**：根据反馈不断优化内容

### 6.3 应急预案

| 风险场景 | 触发条件 | 应急措施 |
|---------|---------|---------|
| App崩溃/音频播放失败 | 用户反馈集中 | 快速热更新、回滚机制 |
| 内容投诉 | 收到家长投诉内容不当 | 立即下架、启动审核 |
| 数据泄露 | 发现安全漏洞 | 立即通知、紧急修复 |
| 审核被拒 | 小程序审核不通过 | 针对性修改、申诉 |

## 7. 与爸妈小故事的升级衔接方案

### 7.1 升级背景

**爸妈小故事现状**
- 已有睡前故事音频播放功能
- 用户基础（需确认具体数据）
- 现有技术架构和代码积累

**升级必要性**
- "音频播放"是儿童睡眠方案的子集
- 可从"工具"升级为"健康方案"
- 扩大用户价值，提升差异化

### 7.2 升级路径设计

```
┌─────────────────────────────────────────────────────────────────┐
│                     爸妈小故事 → 儿童睡眠综合解决方案            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Phase 1: 功能整合（第1-2个月）                                   │
│  ┌──────────────────┐    ┌──────────────────┐                   │
│  │   爸妈小故事      │    │   睡眠方案       │                   │
│  │   现有音频功能    │ +  │   睡前仪式引导   │  = MVP核心       │
│  └──────────────────┘    └──────────────────┘                   │
│                                                                  │
│  Phase 2: 品牌升级（第3-4个月）                                   │
│  ┌──────────────────────────────────────────┐                   │
│  │           新品牌：儿童睡眠综合解决方案                    │   │
│  │   - 独立小程序入口（可共存）                            │   │
│  │   - 爸妈小故事用户引导迁移                               │   │
│  │   - 数据打通，用户无感升级                               │   │
│  └──────────────────────────────────────────┘                   │
│                                                                  │
│  Phase 3: 能力扩展（第5-6个月+）                                  │
│  ┌──────────────────────────────────────────┐                   │
│  │   - 睡眠追踪、家长指导、CBT-I方案                        │   │
│  │   - 新会员体系、差异化定价                               │   │
│  │   - 爸妈小故事作为子模块/内容库保留                      │   │
│  └──────────────────────────────────────────┘                   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 7.3 技术衔接方案

#### 方案一：独立小程序（新推荐）
| 选项 | 说明 | 优势 | 劣势 |
|------|------|------|------|
| 新建独立小程序 | 全新项目架构 | 架构更优、品牌独立 | 用户需重新获取 |
| 爸妈小故事引导页 | 现有小程序增加引导入口 | 流量复用 | 增加爸妈小故事复杂度 |

#### 方案二：原小程序升级
| 选项 | 说明 | 优势 | 劣势 |
|------|------|------|------|
| 整体升级 | 爸妈小故事直接升级为新品牌 | 用户零迁移 | 审核风险、用户体验变化 |
| 模块新增 | 爸妈小故事内新增"睡眠方案"模块 | 渐进式 | 两套逻辑并行复杂 |

**推荐方案：新建独立小程序**
- 品牌定位更清晰
- 技术架构更灵活
- 爸妈小故事作为内容合作方

### 7.4 数据迁移方案

| 数据类型 | 迁移方式 | 说明 |
|---------|---------|------|
| 用户账号 | 微信UnionID关联 | 同一主体下可打通 |
| 音频播放记录 | 历史数据导出+导入 | 确保连续性 |
| 收藏内容 | 内容ID映射 | 保留收藏关系 |
| 会员权益 | 独立新建 | 避免复杂度 |

### 7.5 用户体验衔接

1. **存量用户引导**
   - 爸妈小故事首页/弹窗引导
   - 私信/消息触达
   - 会员权益同步说明

2. **体验一致性**
   - 保持相似设计语言
   - 核心功能操作习惯一致
   - 品牌调性延续

3. **价值提升感知**
   - 明确告知升级带来的新价值
   - 存量用户专属优惠/权益

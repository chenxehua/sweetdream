## 1. 设计概述

### 1.1 技术选型

| 项目 | 选型 | 说明 |
|------|------|------|
| **数据库类型** | MongoDB文档数据库 | 微信云开发内置 |
| **优势** | 免运维、高可用、自动扩展 | 适合小程序场景 |
| **替代方案** | MySQL + 云服务器 | 后期用户量增大后 |

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **规范化优先** | MVP阶段适度冗余，简化查询 |
| **隐私保护** | 儿童数据特殊处理，加密存储 |
| **可扩展** | 预留扩展字段 |
| **高效查询** | 合理索引设计 |

### 1.3 数据类型对照

| MongoDB类型 | JavaScript类型 | 说明 |
|-------------|---------------|------|
| String | string | 字符串 |
| Number | number | 数字 |
| Boolean | boolean | 布尔 |
| Object | object | 对象 |
| Array | array | 数组 |
| Date | Date | 日期 |
| GeoPoint | object | 地理位置 |

## 5. 索引设计

### 5.1 索引列表

```javascript
// 用户表索引
db.user.createIndex({ "openid": 1 }, { unique: true })
db.user.createIndex({ "unionid": 1 }, { sparse: true })
db.user.createIndex({ "status": 1, "vip_status": 1 })

// 孩子档案表索引
db.child_profile.createIndex({ "user_id": 1 })
db.child_profile.createIndex({ "status": 1 })

// 睡眠记录表索引
db.sleep_record.createIndex({ "user_id": 1, "record_date": 1 }, { unique: true })
db.sleep_record.createIndex({ "child_id": 1, "record_date": -1 })

// 音频内容表索引
db.audio_content.createIndex({ "category": 1, "status": 1 })
db.audio_content.createIndex({ "age_range.min": 1, "age_range.max": 1 })
db.audio_content.createIndex({ "is_vip_only": 1, "status": 1 })
db.audio_content.createIndex({ "play_count": -1 })

// 仪式模板表索引
db.ritual_template.createIndex({ "status": 1, "sort_order": 1 })
db.ritual_template.createIndex({ "age_range.min": 1, "age_range.max": 1 })

// 会员订单表索引
db.order.createIndex({ "order_no": 1 }, { unique: true })
db.order.createIndex({ "user_id": 1, "status": 1 })
db.order.createIndex({ "payment.status": 1, "create_time": -1 })

// 收藏记录表索引
db.favorite.createIndex({ "user_id": 1, "content_id": 1 }, { unique: true })
```

### 5.2 查询优化建议

| 查询场景 | 优化方案 |
|---------|---------|
| 查询用户的孩子列表 | user_id索引 |
| 查询日期范围睡眠记录 | child_id + record_date复合索引 |
| 筛选年龄合适的内容 | age_range索引 |
| 查询VIP专属内容 | is_vip_only + status索引 |

## 6. 数据安全与隐私设计

### 6.1 儿童数据保护原则

| 原则 | 说明 | 实施措施 |
|------|------|---------|
| **知情同意** | 明确告知收集目的 | 隐私政策弹窗、首次登录告知 |
| **最小化收集** | 仅收集必要数据 | 功能导向设计，避免过度收集 |
| **家长控制** | 家长完全掌控数据 | 家长账号关联、权限验证 |
| **安全保障** | 保护数据安全 | 加密存储、访问控制 |
| **删除权** | 支持删除数据 | 设置中提供删除入口 |

// 内容访问控制示例
function canAccessContent(user, content) {
  // 公开内容人人可访问
  if (!content.is_vip_only) return true;
  
  // 检查VIP权限
  if (user.vip_status !== "none") return true;
  
  return false;
}

// 数据访问控制示例
function canAccessChildData(userId, childId) {
  // 查询孩子档案
  const child = db.child_profile.findById(childId);
  
  // 仅家长本人可访问
  return child.user_id === userId;
}
